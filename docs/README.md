# Documentación de DayLog

Esta carpeta contiene la documentación completa del proyecto DayLog.

## Estructura de la documentación

- **[Arquitectura](./architecture/)**: Diagramas y decisiones de arquitectura del proyecto
- **[API](./api/)**: Documentación detallada de la API REST

## Cómo usar esta documentación

Esta documentación está diseñada para ser fácil de navegar y entender. Cada sección está organizada en archivos Markdown que pueden visualizarse directamente en GitHub o cualquier visor de Markdown.

## Mantenimiento de la documentación

Para mantener esta documentación actualizada, por favor sigue estas prácticas:

1. Actualiza la documentación cada vez que implementes nuevas características
2. Usa diagramas y ejemplos para explicar conceptos complejos
3. Sigue un estilo consistente en toda la documentación
4. Revisa periódicamente la documentación para asegurar su precisión
